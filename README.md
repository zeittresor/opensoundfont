# FM/GM Open SoundFont

A compact, experimental **General MIDI compatible SoundFont 2 (`.sf2`)** built from deterministic **FM synthesis** samples.

The goal is not to imitate a large acoustic sample library. The goal is a complete GM-mapped SoundFont with a clear FM/digital character that can be used for MIDI playback, retro game music, demos, experiments, and further sound-design work.

> Format: SoundFont 2 (`.sf2`).  
> License: GNU GPLv3, see [License](#license).
> Repository: https://github.com/zeittresor/opensoundfont.

## SoundFont details

- Format: SoundFont 2 / `.sf2`
- Sample format: 44.1 kHz, 16-bit, mono
- Melodic GM programs: 0-127
- Drum notes: GM Standard Drum Kit, notes 35-81
- Synthesis: deterministic FM/additive synthesis generated in Python
- External samples: none
- License: GNU GPLv3, SPDX: `GPL-3.0-only`

## Compatibility banks

The SoundFonts include these compatibility aliases without duplicating sample data:

- Bank `0`: General MIDI melodic programs 0-127
- Selected common GS/XG/GM2 melodic fallback banks: `1-8`, `12`, `14`, `16`, `24`, `25`, `27`, `28`, `32`, `40`, `48`, `56`, `64-72`, `80`, `96-101`, `120`, `121`, `127`, `8192`, `15360`, `15488`
- Bank `128`: drum-kit aliases for programs 0-127
- Bank `8192`: XG-style fallback alias bank
- Banks `15360` and `15488`: GS/GM2-style fallback alias banks
- Banks `16128` and `16256`: XG drum fallback banks


Because every sound is generated by FM recipes, acoustic instruments are **FM-flavoured interpretations**, not realistic multi-sampled emulations.

## Quick start with FluidSynth

### Play a MIDI file directly

```bash
fluidsynth -i FM_GM_SoundFont_v0_1.sf2 your_song.mid
```

### Render a MIDI file to WAV

```bash
fluidsynth -ni FM_GM_SoundFont_v0_1.sf2 your_song.mid -F render.wav -r 44100
```

### Render the included demo MIDI

```bash
fluidsynth -ni FM_GM_SoundFont_v0_1.sf2 FM_GM_SoundFont_v0_1_demo.mid -F render.wav -r 44100
```

### Useful FluidSynth notes

- `-i` starts FluidSynth without the interactive shell.
- `-n` disables reading user configuration files.
- `-F render.wav` writes the rendered audio to a WAV file.
- `-r 44100` sets the render sample rate to 44.1 kHz.
- For realtime playback on Windows, install a FluidSynth build that includes an audio backend such as WASAPI, DirectSound, SDL, or PortAudio.

## Quick start with VLC

VLC can play MIDI files only if its build includes MIDI synthesis support and a SoundFont is configured. Some VLC packages do not include the needed FluidSynth/MIDI module.

### VLC GUI setup

1. Open **VLC**.
2. Go to **Tools → Preferences**.
3. At the bottom, switch **Show settings** to **All**.
4. Open **Input / Codecs → Audio codecs → FluidSynth**.
5. Set **SoundFont file** to `FM_GM_SoundFont_v0_1.sf2`.
6. Save the preferences.
7. Open a `.mid` file in VLC.

### VLC command-line example

Depending on the VLC build, this may work:

```bash
vlc --soundfont="FM_GM_SoundFont_v0_1.sf2" your_song.mid
```

If VLC still refuses to play MIDI, the installed VLC build probably lacks the FluidSynth/MIDI codec module. In that case, use FluidSynth directly or install a VLC build/package with FluidSynth support.

## Using it in DAWs, notation software, and other tools

Typical workflow:

1. Load `FM_GM_SoundFont_v0_1.sf2` as a SoundFont instrument.
2. Select **Bank 0** for melodic GM instruments.
3. Select **Bank 128 / Program 0** for drums, if your host exposes bank selection.
4. Use MIDI channel 10 for GM drums.

Potentially useful programs:

- Polyphone, for inspecting/editing the `.sf2`.
- FluidSynth, for playback/rendering.
- MuseScore, LMMS, Qtractor, Ardour, Reaper, or any host that can load SF2 directly or through a SoundFont plugin.

## Synthesis design

Each melodic preset uses one generated root sample at C4 with SoundFont pitch transposition. Some presets are looped, others are one-shot/decay sounds.

The melodic FM core uses:

```text
carrier = sin(phase * c + idx1 * sin(phase * m1 + idx2 * sin(phase * m2)) + feedback * sin(phase * c))
optional second carrier = sin(phase * c2 + idx1 * 0.55 * sin(phase * (m2 + 0.5)))
output = carrier mixed with optional second carrier via mix2
```

### FM recipe parameters

| Parameter | Meaning |
|---|---|
| `c` | Main carrier ratio. Usually `1.0` for pitch-stable melodic sounds. |
| `m1` | First modulator frequency ratio. |
| `idx1` | Main FM modulation index / brightness amount. |
| `m2` | Second modulator ratio used to shape the first modulator. |
| `idx2` | Second-level modulation index. |
| `c2` | Optional second carrier ratio. |
| `mix2` | Blend amount for the optional second carrier. |
| `feedback` | Simple feedback-like phase term for sharper or more aggressive timbres. |
| `attack_bright` | Brightness multiplier at note attack. |
| `sustain_bright` | Brightness multiplier for sustain/loop or decay tail. |
| `loop` | Whether the generated sample has a sustain loop. |
| `attack_cycles` | Number of cycles used for the generated looped attack segment. |
| `loop_cycles` | Number of cycles used for the generated periodic loop segment. |
| `seconds` | Duration for unlooped/decay samples. |
| `decay` | Exponential decay strength for unlooped samples. |
| `attack` | Attack time used for unlooped samples. |
| `release` | SoundFont volume-envelope release value used at instrument-zone level. |
| `noise` | Small transient noise layer, used mainly for attack/percussion texture. |

### SoundFont generator parameters used

| Generator | Value / role |
|---|---|
| `keyRange` | Melodic presets cover keys `0–127`; drum zones cover one key per drum note. |
| `attackVolEnv` | `0.001 s` for all instrument zones. |
| `releaseVolEnv` | Per-preset melodic release from the FM recipe; `0.08 s` for drums. |
| `sampleModes` | `1` for looped melodic samples, `0` for one-shot samples/drums. |
| `scaleTuning` | `0` for drum zones so drum samples do not transpose by pitch. |
| `overridingRootKey` | `60` for melodic presets; original MIDI note for drum samples. |
| `sampleID` | References the generated sample assigned to the preset or drum note. |

## Troubleshooting

### FluidSynth says it cannot open the SoundFont

Check that the `.sf2` path is correct. On Windows, quote paths that contain spaces:

```bat
fluidsynth -ni "C:\Path With Spaces\FM_GM_SoundFont_v0_1.sf2" "song.mid" -F "render.wav" -r 44100
```

### VLC does not play MIDI

VLC requires a configured SoundFont and a build that contains MIDI/FluidSynth support. If the FluidSynth section is missing in VLC preferences, use FluidSynth directly or install a VLC package that includes the module.

### Drums are silent or play as melodic notes

Use MIDI channel 10 for percussion. If your sequencer exposes banks, select **Bank 128 / Program 0** for the drum track.

### Instruments sound synthetic

That is expected. The SoundFont deliberately uses FM synthesis for all timbres. It is a retro/digital GM bank, not a realistic orchestral/acoustic bank.

## Known limitations of v0.1

- One generated root sample per melodic preset; no velocity layers and no multi-sampling across the keyboard.
- Acoustic instruments are approximate FM interpretations.
- No full General MIDI Level 2 bank set; only GM melodic bank 0 plus a standard drum kit.
- No dedicated stereo samples.
- No manually tuned per-instrument effects such as reverb/chorus presets.
- Drum kit covers the common GM Standard Kit note range `35–81`.

## Rebuilding the SoundFont

The SoundFont was generated programmatically. If the build script is included in the repository, it can be used to regenerate the `.sf2`, demo MIDI, and GM map from the deterministic FM recipes.

Typical repository layout:

```text
.
├── README.md
├── LICENSE
├── FM_GM_SoundFont_v0_1.sf2
├── FM_GM_SoundFont_v0_4_HQ.sf2
├── FM_GM_SoundFont_v0_1_demo.mid
└── FM_GM_SoundFont_v0_1_GM_Map.txt
```

Typical rebuild command:

```bash
python build_fm_gm_sf2.py
```

Generated output names may vary by version. The distributed `.sf2` file is the ready-to-use SoundFont; rebuilding is only needed for development or modification.
